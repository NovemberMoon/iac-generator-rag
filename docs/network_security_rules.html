Сетевая безопасность инфраструктуры

Данные правила обязательны к применению при генерации любого инфраструктурного кода.

1. Управление портами (Security Groups / Firewalls)

Категорически запрещено открывать порт 22 (SSH) и порт 3389 (RDP) для всего интернета (адрес 0.0.0.0/0). Доступ к этим портам разрешен только из внутренней сети 10.0.0.0/24.

Разрешается открывать порты 80 и 443 для всего интернета (0.0.0.0/0) только для веб-серверов.

2. Привязка групп безопасности (КРИТИЧНО)

Любые созданные правила брандмауэра (группы безопасности) ОБЯЗАТЕЛЬНО должны быть логически привязаны к сетевым интерфейсам соответствующих виртуальных машин, используя корректный синтаксис целевого провайдера.

3. Изоляция баз данных

Серверы баз данных (PostgreSQL, MySQL, Redis) никогда не должны получать публичный IP-адрес. Используй специфичные для целевого облака методы изоляции.
