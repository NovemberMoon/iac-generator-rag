# Внутренний регламент написания IaC (ООО "НоябрьКун")

## 1. Строгая политика тегирования (Mandatory Labels)

АБСОЛЮТНО ВСЕ создаваемые ресурсы в Terraform (виртуальные машины, сети, диски), которые поддерживают блок labels, ОБЯЗАНЫ содержать следующие корпоративные метки:

* department = "cybernetics"  
* cost_center = "777"  
* managed_by = "rag-generator"

Если этих меток нет, инфраструктура не пройдет проверку безопасности (Security Gate).

Пример правильного блока:

```
labels = {  
  department  = "cybernetics"  
  cost_center = "777"  
  managed_by  = "rag-generator"  
}
```

## 2. Соглашение об именовании (Naming Convention)

Любая виртуальная машина (yandex_compute_instance) должна обязательно иметь префикс vm-corp- в своем имени.

Например: name = "vm-corp-ubuntu-server".

## 3. Настройка пользователя по умолчанию

При развертывании виртуальных машин запрещено использовать стандартного пользователя ubuntu. Вместо него в блоке metadata (в user-data или ssh-keys) должен указываться пользователь corp-admin.
