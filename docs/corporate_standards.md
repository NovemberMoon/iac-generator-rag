# Внутренний регламент инфраструктуры (ООО "НоябрьКун")

## 1. Строгая политика тегирования (Mandatory Labels)

АБСОЛЮТНО ВСЕ создаваемые ресурсы (виртуальные машины, сети, диски), которые поддерживают тегирование/метки в выбранном инструменте, ОБЯЗАНЫ содержать следующие корпоративные метки:

- department = "cybernetics"
- cost_center = "777"
- managed_by = "rag-generator"

Если этих меток нет, инфраструктура не пройдет проверку безопасности.

## 2. Соглашение об именовании (Naming Convention)

Любая виртуальная машина или сервер должны обязательно иметь префикс `vm-corp-` в своем имени. Например: `vm-corp-ubuntu-server`.

## 3. Настройка пользователя по умолчанию

При развертывании и настройке виртуальных машин запрещено использовать стандартных пользователей (ubuntu, ec2-user, root). Всегда должен настраиваться и использоваться пользователь `corp-admin`.
